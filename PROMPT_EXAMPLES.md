# 🎯 Gemini AI 프롬프트 예시 모음

Gemini AI를 사용하여 ML 파이프라인을 자동 생성할 때 유용한 프롬프트 예시입니다.

## 🌸 분류 (Classification)

### 기본 분류
```
아이리스 데이터셋으로 꽃의 종류를 분류하는 랜덤 포레스트 모델을 만들어주세요
```

### 상세 분류
```
타이타닉 생존 예측 모델을 만들어주세요.
- 데이터 분할: 80% 학습, 20% 테스트
- 전처리: StandardScaler 사용
- 알고리즘: RandomForest 분류기
- 평가: accuracy, precision, recall
```

### SVM 분류
```
유방암 진단 데이터로 악성/양성을 분류하는 SVM 모델 만들기
데이터를 MinMax 스케일링하고 rbf 커널 사용
```

### 로지스틱 회귀
```
고객 이탈 예측을 위한 로지스틱 회귀 모델
- 데이터 분할: 75/25
- 정규화: L2 (Ridge)
- 평가 지표: ROC-AUC 포함
```

### 신경망 분류
```
손글씨 숫자(MNIST)를 분류하는 신경망 모델
- 구조: 128-64-10 층
- 활성화 함수: ReLU
- 옵티마이저: Adam
- 에포크: 50
```

## 📈 회귀 (Regression)

### 기본 선형 회귀
```
주택 가격을 예측하는 선형 회귀 모델을 만들어주세요
```

### 학생 점수 예측 🆕
```
중간고사 점수로 기말고사 점수를 예측하는 모델 만들기
- Linear Regression 사용
- 80:20 train/test 분할
- R², MAE, MSE, RMSE로 평가
- 새로운 학생의 점수 예측 기능 포함
```

### Ridge 회귀
```
보스턴 주택 가격 예측 모델
- Ridge 회귀 사용 (alpha=1.0)
- 데이터 스케일링 필수
- R2 스코어로 평가
```

### RandomForest 회귀
```
중고차 가격 예측을 위한 랜덤 포레스트 회귀 모델
- 100개 트리 사용
- 상위 15개 특성만 선택
- RMSE와 MAE로 평가
```

### Lasso 회귀
```
와인 품질 점수 예측 Lasso 회귀 모델
- alpha=0.1
- 특성 선택 자동화
- 교차 검증 포함
```

### 다항 회귀
```
시간에 따른 주가 예측 모델
- 2차 다항 특성 생성
- Ridge 회귀 적용
- 80:20 분할
```

## 🔧 전처리 중심

### 스케일링
```
데이터를 StandardScaler로 정규화하고 
RandomForest 분류기로 학습시켜주세요
```

### 특성 선택
```
1000개 특성 중 상위 50개만 선택하여 
SVM으로 분류하는 파이프라인
```

### 다단계 전처리
```
1. 데이터 로드
2. 75:25 분할
3. MinMax 스케일링 (0-1 범위)
4. 분산 기반 특성 선택 (상위 20개)
5. Gradient Boosting 분류
6. 정확도, F1 스코어 평가
```

## 🎛️ 하이퍼파라미터 튜닝

### GridSearch
```
SVM 모델의 최적 파라미터를 찾아주세요
- GridSearch 사용
- C와 gamma 탐색
- 5-fold 교차 검증
```

### RandomSearch
```
RandomForest의 최적 설정 탐색
- RandomizedSearch 사용
- n_estimators, max_depth 조정
- 3-fold CV
```

## 🧪 복합 파이프라인

### 앙상블 모델
```
여러 분류 모델을 비교하는 파이프라인
1. 데이터 분할 및 스케일링
2. 3가지 모델 학습 (Logistic, RF, SVM)
3. 각 모델 성능 평가 및 비교
```

### End-to-End 파이프라인
```
완전한 ML 프로젝트 파이프라인 구축
1. CSV 데이터 로드 (iris.csv)
2. 80:20 train/test 분할 (random_state=42)
3. Standard 스케일링
4. K-Best로 특성 3개 선택
5. RandomForest 분류 (100 트리)
6. 하이퍼파라미터 튜닝 (GridSearch, 3-fold)
7. 최종 모델 평가 (accuracy, precision, recall, f1)
8. 테스트 데이터로 예측
```

### 시계열 예측
```
시계열 데이터로 미래 값 예측
1. 시계열 데이터 로드
2. 시간 기반 train/test 분할 (최근 20% 테스트)
3. 특성 엔지니어링 (lag features)
4. RandomForest 회귀
5. RMSE, MAE 평가
```

## 💡 프롬프트 작성 팁

### ✅ 효과적인 프롬프트
- **명확한 목적**: "분류", "회귀", "예측" 등 명시
- **구체적인 수치**: "80:20 분할", "100 에포크" 등
- **알고리즘 지정**: "RandomForest", "SVM", "Neural Network"
- **단계별 설명**: 전처리 → 학습 → 평가 순서 명시

### ❌ 비효율적인 프롬프트
- "좋은 모델 만들어줘" (목적 불명확)
- "데이터 분석" (ML 파이프라인이 아님)
- "최고의 결과" (구체적 방법 없음)

## 🎨 창의적 활용

### 도메인 특화
```
의료 이미지 분류 모델
- 합성곱 신경망 (3층)
- 배치 정규화 포함
- Dropout 0.5
- 200 에포크
```

### 비즈니스 분석
```
마케팅 캠페인 효과 예측
- 고객 세그먼트별 분할
- Logistic Regression
- 비용-효과 분석 포함
```

### 연구 프로토타입
```
새로운 알고리즘 비교 실험
1. 동일한 데이터셋
2. 5가지 분류기 (LR, RF, SVM, KNN, GB)
3. 10-fold 교차 검증
4. 성능 지표 표로 정리
```
