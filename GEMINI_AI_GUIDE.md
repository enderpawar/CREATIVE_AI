# 🤖 Gemini AI 파이프라인 자동 생성 가이드

## 개요
Gemini AI를 활용하여 자연어 프롬프트로부터 ML 파이프라인을 자동으로 생성할 수 있습니다.

## API 키 설정

### 방법 1: UI에서 설정 (권장)
1. 로직 에디터 페이지 우측 상단의 "🤖 AI 파이프라인 생성" 섹션을 찾습니다
2. "Gemini API 키" 입력란에 API 키를 입력합니다
3. "저장" 버튼을 클릭합니다

### 방법 2: 환경변수 설정
1. `.env` 파일을 생성합니다 (`.env.example` 참고)
2. `VITE_GEMINI_API_KEY=your_api_key_here` 형식으로 입력합니다
3. 개발 서버를 재시작합니다

### API 키 발급
- [Google AI Studio](https://aistudio.google.com/app/apikey)에서 무료로 발급받을 수 있습니다
- Google 계정으로 로그인 후 "Create API Key" 클릭
- 발급된 키를 복사하여 사용

## 사용 방법

### 1. 프롬프트 작성
우측 사이드바의 텍스트 영역에 원하는 ML 파이프라인을 자연어로 설명합니다.

**예시:**
- "아이리스 데이터셋으로 꽃의 종류를 분류하는 랜덤 포레스트 모델을 만들어주세요"
- "주택 가격을 예측하는 선형 회귀 모델을 만들어주세요. 데이터는 80:20으로 분할해주세요"
- "손글씨 숫자를 분류하는 신경망 모델을 만들어주세요. 3개 층으로 구성하고 100 에포크로 학습해주세요"

### 2. 파이프라인 생성
"✨ AI로 파이프라인 생성하기" 버튼을 클릭합니다.

### 3. 결과 확인
- 생성된 노드들이 자동으로 캔버스에 배치됩니다
- 노드 간 연결도 자동으로 생성됩니다
- 각 노드의 파라미터도 프롬프트에 따라 설정됩니다

## 프롬프트 작성 팁

### ✅ 좋은 프롬프트 예시
```
"타이타닉 생존 예측을 위한 분류 모델을 만들어주세요.
- 데이터 분할: 80% 학습, 20% 테스트
- 스케일링: StandardScaler 사용
- 모델: RandomForest 분류기
- 평가 지표: accuracy, precision, recall"
```

### ❌ 피해야 할 프롬프트
```
"ML 모델 만들어줘"  (너무 모호함)
"좋은 모델"         (구체적인 요구사항 없음)
```

### 포함하면 좋은 정보
1. **데이터**: 어떤 데이터를 사용할지
2. **작업 유형**: 분류, 회귀, 군집화 등
3. **알고리즘**: 사용할 모델 (RandomForest, SVM, Neural Network 등)
4. **파라미터**: 특정 설정값 (test_size, epochs, layers 등)
5. **전처리**: 스케일링, 특성 선택 등

## 생성된 파이프라인 수정

생성된 파이프라인은 자유롭게 수정할 수 있습니다:
- **노드 추가**: 좌측 사이드바에서 드래그하여 추가
- **노드 삭제**: 노드 선택 후 Delete 키
- **연결 수정**: 기존 연결 삭제 후 새로 연결
- **파라미터 조정**: 각 노드를 클릭하여 값 수정

## 예시 프롬프트 모음

### 분류 (Classification)
```
"아이리스 데이터로 꽃 종류를 분류하는 SVM 모델을 만들어주세요"
"스팸 메일 필터링을 위한 로지스틱 회귀 모델"
"이미지 분류를 위한 신경망 (3층, ReLU 활성화)"
```

### 회귀 (Regression)
```
"주택 가격 예측 회귀 모델 (Ridge 정규화 포함)"
"선형 회귀로 매출 예측 모델 만들기"
"RandomForest 회귀로 기온 예측"
```

### 전처리 포함
```
"데이터를 MinMax 스케일링하고 RandomForest로 분류하기"
"상위 10개 특성만 선택하여 SVM 모델 학습"
"GridSearch로 최적 하이퍼파라미터 찾고 모델 평가하기"
```

## 문제 해결

### API 키 오류
- API 키가 올바른지 확인
- Google AI Studio에서 키 활성화 상태 확인
- 브라우저 콘솔에서 오류 메시지 확인

### 파이프라인 생성 실패
- 프롬프트를 더 구체적으로 작성
- 사용 가능한 노드 타입 확인 (dataLoader, classifier, regressor 등)
- 네트워크 연결 확인

### 노드가 캔버스에 나타나지 않음
- 에디터가 준비될 때까지 대기
- 페이지 새로고침 후 재시도
- 브라우저 콘솔에서 오류 확인

## 제한사항

1. **노드 타입**: 현재 지원되는 10가지 ML 노드만 생성 가능
2. **연결 규칙**: 일부 노드는 특정 방식으로만 연결 가능
3. **API 제한**: Gemini API의 요청 제한 및 할당량 적용
4. **프롬프트 길이**: 너무 긴 프롬프트는 잘릴 수 있음
